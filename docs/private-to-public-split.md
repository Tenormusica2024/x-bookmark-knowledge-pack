# Private to Public Split

## Principle

The private repo remains the upstream experimentation track.

The public repo becomes the generalized distribution track.

---

## Keep in private

- real production bookmark data
- personal paths / local machine assumptions
- private operational scripts
- unstable experiments
- personal scheduling / deployment glue

---

## Move or recreate in public

- generalized viewer
- packaging logic
- sample data
- public documentation
- safe defaults
- clear setup instructions

---

## Migration style

Recommended workflow:

1. experiment in private
2. verify usefulness
3. refactor for portability
4. move only the cleaned public-safe subset

This repo should receive only what can be explained and reused publicly.
