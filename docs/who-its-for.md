# Who It's For

## Good fit

This project is a good fit if you want to:

- keep X bookmark reuse local-first
- turn saved posts into a reusable knowledge asset
- browse them as HTML and also feed them to AI agents
- keep an extracted bookmark DB / export process as the upstream source of truth
- refresh the bundle on a schedule without building a full SaaS stack

---

## Probably not a fit

This project is probably **not** the best fit if you want:

- a polished cloud bookmark SaaS with sync and collaboration first
- a generic bookmarking product for every kind of web page
- a tool that directly handles all upstream collection by itself
- a zero-setup product with no scheduler / local file concepts

---

## Positioning in one sentence

This repo is best understood as a **local knowledge-pack layer on top of an extracted X bookmark source**, not as a general bookmark manager.
